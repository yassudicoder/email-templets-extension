01 — Components

Four parts, four jobs

The extension isn't one big program. It's four small pieces, each responsible for one thing. Here's each one in everyday terms.

Content scripts in-page

Run inside the Gmail or LinkedIn tab while you work. They watch for your hotkey, show the picker, and place text at your cursor. The hands-on helpers sitting on the page with you.

Service worker background

A background process that routes messages and runs quiet jobs. It sleeps when idle and wakes on demand. The dispatcher in the back office.

Local storage on your device

Where every template and setting is kept — in your own browser, on your own machine. Nothing here is uploaded on its own. A filing cabinet at home.

Options page the dashboard

The full management screen — write templates, sort them into folders, switch themes, empty the trash. The control room.

02 — Topology

04 — Release v0.2.0 · shipped

From one flat list to an organised library

Previously every template sat in a single list. We restructured storage to support folders, favourites, and a recoverable trash, and rebuilt the dashboard around them.

Folders

Group templates by purpose. Create, rename, delete. Deleting a folder never deletes its templates — they return to “All”.

Recoverable trash

Deleting moves a template to Trash, not oblivion. Restore it, or delete permanently when you’re sure.

Three-pane dashboard

Folders left, template cards centre, editor right — the layout of a modern mail client.

Preview

See a template rendered, with fill-in fields like {first_name} highlighted, before using it.

Migration
Existing templates carried over automatically with no data loss; they appear under “All Templates” until filed.

05 — Release v0.3.0 · next

Optional, opt-in analytics

To learn which features are used, we added usage analytics — implemented as a setting that is off by default. Nothing is collected unless the user turns it on.

  • Identical to a build with no analytics at all.

How it's enforced — a single checkpoint

Every event is routed through the service worker, which checks the setting before any network request. It's the only exit, so nothing bypasses the check.

Because there is one exit that always checks the setting first, “off” means precisely zero data leaves — confirmed under test (0 requests off, 1 on, with no content in the payload).
